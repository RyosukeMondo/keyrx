# keyrx_ffi_macro

Procedural macro for generating FFI code from contract definitions.

## Overview

`keyrx_ffi_macro` generates type-safe FFI wrapper functions from JSON contract files at compile time. This ensures:

- FFI function signatures match contracts exactly
- Consistent error handling across all FFI functions
- Panic safety at FFI boundaries
- Automatic C string parsing and JSON serialization

## Quick Start

### 1. Define a Contract

Create a contract file at `src/ffi/contracts/{domain}.ffi-contract.json`:

```json
{
    "version": "1.0.0",
    "domain": "myservice",
    "description": "My service FFI interface",
    "protocol_version": 1,
    "functions": [
        {
            "name": "get_value",
            "description": "Get a value by key",
            "parameters": [
                {
                    "name": "key",
                    "type": "string",
                    "description": "The key to look up",
                    "required": true
                }
            ],
            "returns": {
                "type": "string",
                "description": "The value"
            }
        }
    ]
}
```

### 2. Implement the Domain

```rust
use keyrx_ffi_macro::keyrx_ffi;

#[keyrx_ffi(domain = "myservice")]
impl MyServiceDomain {
    fn get_value(key: String) -> Result<String, String> {
        // Your implementation here
        Ok(format!("value for {}", key))
    }
}

struct MyServiceDomain;
```

### 3. Generated FFI Functions

The macro generates `extern "C"` functions for each contract function:

```rust
// Generated by the macro:
#[no_mangle]
pub unsafe extern "C" fn keyrx_myservice_get_value(
    error: *mut *mut c_char,
    key: *const c_char,
) -> *const c_char {
    // ... implementation with panic catching, error handling, etc.
}
```

## Contract Schema

### Root Object

| Field | Type | Description |
|-------|------|-------------|
| `version` | string | Contract version (semver) |
| `domain` | string | Domain name (used in function naming) |
| `description` | string | Human-readable description |
| `protocol_version` | number | Protocol version (currently 1) |
| `functions` | array | List of function definitions |
| `types` | object | Optional type definitions |

### Function Definition

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Function name (used for impl method matching) |
| `rust_name` | string? | Optional custom FFI function name |
| `description` | string | Human-readable description |
| `parameters` | array | List of parameters |
| `returns` | object | Return type definition |
| `errors` | array | Possible error codes |

### Parameter Definition

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Parameter name |
| `type` | string | Type name (see supported types) |
| `description` | string | Human-readable description |
| `required` | boolean | Whether the parameter is required (default: true) |

### Supported Types

| Contract Type | Rust FFI Type | Description |
|---------------|---------------|-------------|
| `string` | `*const c_char` | C string (null-terminated UTF-8) |
| `int32`, `int` | `i32` | 32-bit signed integer |
| `uint8` | `u8` | 8-bit unsigned integer |
| `uint32` | `u32` | 32-bit unsigned integer |
| `uint64` | `u64` | 64-bit unsigned integer |
| `float64`, `float`, `double` | `f64` | 64-bit floating point |
| `bool`, `boolean` | `bool` | Boolean |
| `void`, `unit` | `()` | No return value |
| `object` | `*const c_char` | JSON-encoded object |
| `array` | `*const c_char` | JSON-encoded array |

## Error Handling

All generated FFI functions accept an error pointer as the first parameter:

```c
// C caller
char* error = NULL;
const char* result = keyrx_myservice_get_value(&error, "key");
if (error != NULL) {
    // Handle error
    printf("Error: %s\n", error);
    keyrx_free_string(error);
} else {
    // Use result
    printf("Result: %s\n", result);
    keyrx_free_string(result);
}
```

The error pointer is set when:
- A Rust `Result::Err` is returned
- A panic occurs in the Rust code
- Parameter parsing fails (e.g., invalid UTF-8)

## Function Naming Convention

FFI functions are named using the pattern:
- Default: `keyrx_{domain}_{function_name}`
- Custom: Specify `rust_name` in the contract

Example:
```json
{
    "name": "list_items",
    "rust_name": "keyrx_config_list_items"  // Custom name
}
```

## Dependencies

The generated code depends on `keyrx_ffi_runtime` for:
- `parse_c_string`: Safe C string to Rust String conversion
- `serialize_to_c_string`: Rust values to C string (via JSON)
- `ffi_wrapper`: Complete FFI wrapper with panic catching

Add to your `Cargo.toml`:
```toml
[dependencies]
keyrx_ffi_macro = { path = "keyrx_ffi_macro" }
keyrx_ffi_runtime = { path = "keyrx_ffi_runtime" }
```

## Compile-Time Contract Loading

The macro loads contracts at compile time using `CARGO_MANIFEST_DIR`. It searches:

1. `{MANIFEST_DIR}/src/ffi/contracts/{domain}.ffi-contract.json`
2. `{MANIFEST_DIR}/../src/ffi/contracts/{domain}.ffi-contract.json`

This allows the macro to work from both the main crate and subcrates.

## Migration Guide

### Before: Manual FFI Implementation

```rust
use std::ffi::{c_char, CString, CStr};

#[no_mangle]
pub unsafe extern "C" fn keyrx_config_get_value(
    error: *mut *mut c_char,
    key: *const c_char,
) -> *const c_char {
    // Manual null check
    if key.is_null() {
        *error = CString::new("key is null").unwrap().into_raw();
        return std::ptr::null();
    }

    // Manual panic catching
    let result = std::panic::catch_unwind(|| {
        // Manual UTF-8 conversion
        let key_str = CStr::from_ptr(key).to_str()?;

        // Business logic
        let value = get_value_impl(key_str)?;

        // Manual result serialization
        Ok(CString::new(value).unwrap().into_raw())
    });

    match result {
        Ok(Ok(ptr)) => ptr,
        Ok(Err(e)) => {
            *error = CString::new(e.to_string()).unwrap().into_raw();
            std::ptr::null()
        }
        Err(_) => {
            *error = CString::new("panic occurred").unwrap().into_raw();
            std::ptr::null()
        }
    }
}
```

### After: Macro-Generated FFI

**Step 1:** Create the contract file `src/ffi/contracts/config.ffi-contract.json`:

```json
{
    "version": "1.0.0",
    "domain": "config",
    "description": "Configuration management",
    "protocol_version": 1,
    "functions": [
        {
            "name": "get_value",
            "rust_name": "keyrx_config_get_value",
            "description": "Get a configuration value",
            "parameters": [
                { "name": "key", "type": "string", "description": "Config key" }
            ],
            "returns": { "type": "string", "description": "Config value" }
        }
    ]
}
```

**Step 2:** Implement the domain with the macro:

```rust
use keyrx_ffi_macro::keyrx_ffi;

#[keyrx_ffi(domain = "config")]
impl ConfigDomain {
    fn get_value(key: String) -> Result<String, String> {
        get_value_impl(&key)
    }
}

struct ConfigDomain;
```

### Migration Checklist

1. **Create contract file** for your domain
2. **Match function names** between contract and impl methods
3. **Update parameter types** to Rust types (macro handles conversion)
4. **Return `Result<T, String>`** from impl methods
5. **Remove manual FFI boilerplate**:
   - No `#[no_mangle]`
   - No `extern "C"`
   - No manual C string handling
   - No panic catching code
6. **Add macro dependencies** to Cargo.toml

### Key Differences

| Aspect | Manual | Macro |
|--------|--------|-------|
| Function signature | `extern "C"` with raw pointers | Normal Rust method |
| Parameter types | `*const c_char`, `i32`, etc. | `String`, `i32`, etc. |
| Return type | `*const c_char` | `Result<T, String>` |
| Error handling | Manual error pointer setting | Automatic |
| Panic safety | Manual `catch_unwind` | Automatic |
| String conversion | Manual `CStr`/`CString` | Automatic |
| Lines of code | ~30+ per function | ~5 per function |

## Example: Complete Implementation

```rust
//! src/ffi/domains/config.rs

use keyrx_ffi_macro::keyrx_ffi;
use std::sync::OnceLock;

/// Global state for the config domain.
fn config_store() -> &'static ConfigStore {
    static STORE: OnceLock<ConfigStore> = OnceLock::new();
    STORE.get_or_init(ConfigStore::new)
}

/// Config domain implementation.
///
/// All methods matching contract functions will have FFI wrappers generated.
#[keyrx_ffi(domain = "config")]
impl ConfigDomain {
    /// List all configuration keys.
    fn list_keys() -> Result<String, String> {
        let keys = config_store().list_keys();
        serde_json::to_string(&keys).map_err(|e| e.to_string())
    }

    /// Get a configuration value by key.
    fn get_value(key: String) -> Result<String, String> {
        config_store()
            .get(&key)
            .ok_or_else(|| format!("Key not found: {}", key))
    }

    /// Set a configuration value.
    fn set_value(key: String, value: String) -> Result<String, String> {
        config_store().set(&key, &value);
        Ok(r#"{"success": true}"#.to_string())
    }

    /// Delete a configuration key.
    fn delete_key(key: String) -> Result<String, String> {
        config_store().delete(&key);
        Ok(r#"{"success": true}"#.to_string())
    }
}

struct ConfigDomain;
```

## Troubleshooting

### "failed to load contract for domain"

The contract file wasn't found. Check:
- File exists at `src/ffi/contracts/{domain}.ffi-contract.json`
- Domain name in macro matches filename
- Running from correct directory

### "expected `domain`, found `...`"

The macro attribute is malformed:
```rust
// Wrong
#[keyrx_ffi(module = "config")]

// Correct
#[keyrx_ffi(domain = "config")]
```

### "keyrx_ffi cannot be applied to trait implementations"

The macro only works on inherent impl blocks:
```rust
// Wrong
#[keyrx_ffi(domain = "config")]
impl MyTrait for ConfigDomain { }

// Correct
#[keyrx_ffi(domain = "config")]
impl ConfigDomain { }
```

### "domain cannot be empty"

Provide a non-empty domain name:
```rust
// Wrong
#[keyrx_ffi(domain = "")]

// Correct
#[keyrx_ffi(domain = "config")]
```
