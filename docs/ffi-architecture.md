# KeyRX FFI Architecture

This document describes the Foreign Function Interface (FFI) architecture for KeyRX, which enables communication between the Rust core and Flutter/Dart UI layer.

## Overview

The KeyRX FFI architecture is built on a trait-based, domain-oriented design that provides:

- **Instance-scoped state management**: Handle-based context system replacing global statics
- **Unified callback system**: Single `EventRegistry` for all callback types
- **Automatic code generation**: `#[ffi_export]` macro generates C-ABI wrappers
- **Type safety**: Compile-time guarantees for FFI exports
- **Panic safety**: All panics caught at FFI boundary
- **Parallel test support**: Multiple FFI contexts can run concurrently

### Architecture Layers

```
┌─────────────────────────────────────────────────────┐
│  Flutter/Dart UI Layer                              │
│  - KeyrxBridge (ui/lib/ffi/bridge.dart)            │
│  - Generated bindings (ui/lib/ffi/generated/)      │
└──────────────────┬──────────────────────────────────┘
                   │ FFI Boundary
┌──────────────────▼──────────────────────────────────┐
│  C-ABI Wrapper Layer (Generated by #[ffi_export])   │
│  - Null checks, UTF-8 validation                    │
│  - Panic catching                                    │
│  - JSON serialization                                │
└──────────────────┬──────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────┐
│  Domain Layer (Implements FfiExportable)            │
│  - DiscoveryFfi, ValidationFfi, EngineFfi, etc.    │
│  - Business logic                                    │
│  - State management via FfiContext                  │
└──────────────────┬──────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────┐
│  Core KeyRX Layer                                    │
│  - Discovery, Validation, Engine modules            │
│  - Driver abstraction                                │
└─────────────────────────────────────────────────────┘
```

## Architecture Components

### 1. FfiContext (core/src/ffi/context.rs)

The `FfiContext` provides instance-scoped state management:

**Purpose**: Replace global `OnceLock<Mutex<...>>` patterns with handle-based state

**Key Features**:
- Unique handle per context (`u64`)
- Thread-safe domain state storage (`Arc<RwLock<>>`)
- Type-safe state access via `TypeId`
- Global registry for handle-to-context mapping

**Example Usage**:
```rust
let mut ctx = FfiContext::new();
let handle = ctx.handle();

// Store domain-specific state
struct DiscoveryState { active: bool }
ctx.set_domain("discovery", DiscoveryState { active: true });

// Retrieve domain state
let state = ctx.get_domain::<DiscoveryState>("discovery");
```

### 2. FfiError and FfiResult (core/src/ffi/error.rs)

Standardized error handling across all FFI exports:

**FfiError Structure**:
```rust
pub struct FfiError {
    pub code: String,      // e.g., "INVALID_INPUT"
    pub message: String,   // Human-readable message
    pub details: Option<serde_json::Value>, // Optional context
}
```

**JSON Serialization Format**:
- Success: `"ok:{...serialized T...}"`
- Error: `"error:{code, message, details?}"`

**Helper Constructors**:
```rust
FfiError::invalid_input("device_id must not be empty")
FfiError::internal("unexpected panic")
FfiError::not_found("device")
FfiError::null_pointer("param_name")
FfiError::invalid_utf8("param_name")
```

### 3. EventRegistry (core/src/ffi/events.rs)

Unified callback registration system replacing per-domain callback functions:

**Callback Signature**:
```c
unsafe extern "C" fn(ptr: *const u8, len: usize)
```

**Event Types**:
```rust
pub enum EventType {
    // Discovery domain
    DiscoveryProgress,
    DiscoveryDuplicate,
    DiscoverySummary,

    // Engine domain
    EngineState,

    // Validation domain
    ValidationProgress,
    ValidationResult,

    // ... additional domains
}
```

**Usage**:
```rust
let registry = EventRegistry::new();

// Register callback
registry.register(EventType::DiscoveryProgress, Some(callback_fn));

// Invoke callback with JSON payload
let data = json!({"stage": "scanning", "progress": 0.5});
registry.invoke(EventType::DiscoveryProgress, &data);

// Unregister
registry.register(EventType::DiscoveryProgress, None);
```

### 4. FfiExportable Trait (core/src/ffi/traits.rs)

Contract for FFI-exportable domain modules:

```rust
pub trait FfiExportable {
    /// Domain name for namespacing (e.g., "discovery", "validation")
    const DOMAIN: &'static str;

    /// Initialize domain state within the context
    fn init(ctx: &mut FfiContext) -> Result<(), FfiError>;

    /// Clean up domain state and release resources
    fn cleanup(ctx: &mut FfiContext);
}
```

**Implementation Pattern**:
```rust
pub struct DiscoveryFfi;

impl FfiExportable for DiscoveryFfi {
    const DOMAIN: &'static str = "discovery";

    fn init(ctx: &mut FfiContext) -> Result<(), FfiError> {
        struct DiscoveryState {
            session: Option<DiscoverySession>,
            active: bool,
        }

        ctx.set_domain(Self::DOMAIN, DiscoveryState {
            session: None,
            active: false,
        });

        Ok(())
    }

    fn cleanup(ctx: &mut FfiContext) {
        // Cancel any active session
        // Release resources
        ctx.remove_domain(Self::DOMAIN);
    }
}
```

### 5. #[ffi_export] Macro (core/ffi-macros/src/lib.rs)

Procedural macro that generates C-ABI wrappers automatically:

**What the Macro Does**:
1. Creates `#[no_mangle] pub extern "C"` wrapper function
2. Adds null checks for pointer parameters
3. Validates UTF-8 for string parameters
4. Catches panics with `std::panic::catch_unwind`
5. Converts panics to `FfiError::internal`
6. Serializes results to JSON format
7. Manages string ownership (returns `*mut c_char`)

**Usage**:
```rust
impl DiscoveryFfi {
    #[ffi_export]
    fn start_discovery(
        device_id: &str,
        rows: u8,
        cols_per_row_json: &str,
    ) -> FfiResult<DiscoveryStartResult> {
        // Business logic here
        Ok(DiscoveryStartResult {
            success: true,
            total_keys: Some(104),
        })
    }
}
```

**Generated Function Signature**:
```c
char* keyrx_start_discovery(
    const char* device_id,
    uint8_t rows,
    const char* cols_per_row_json
);
```

## Core Concepts

### Lifecycle Management

Each FFI domain follows this lifecycle:

```
1. Context Creation
   ↓
2. Domain Initialization (FfiExportable::init)
   ↓
3. Domain Methods Invoked via FFI
   ↓
4. Domain Cleanup (FfiExportable::cleanup)
   ↓
5. Context Disposal
```

**From Dart**:
```dart
// Create context
final handle = keyrxCreateContext();

// Initialize domain
keyrxInitDiscovery(handle);

// Use domain
final result = keyrxStartDiscovery(handle, deviceId, rows, cols);

// Cleanup
keyrxCleanupDiscovery(handle);
keyrxDestroyContext(handle);
```

### Handle-Based State

Handles are opaque `u64` identifiers that:
- Uniquely identify each FFI context instance
- Enable parallel test execution (no shared global state)
- Provide safe access across FFI boundary
- Are managed by `FfiContextRegistry`

**Handle Flow**:
```
Rust: FfiContext::new() → generates handle → stores in registry
  ↓
FFI: Returns handle to Dart
  ↓
Dart: Stores handle, passes to all FFI calls
  ↓
FFI: Looks up context by handle in registry
  ↓
Rust: Accesses domain state from context
```

### Thread Safety

All FFI components are thread-safe:

- `FfiContext`: Domain storage uses `Arc<RwLock<>>`
- `EventRegistry`: Callback map uses `Arc<RwLock<HashMap>>`
- `FfiContextRegistry`: Global registry uses `RwLock<HashMap>`

**Concurrent Access Pattern**:
```rust
// Multiple threads can safely access domain state
let guard = ctx.get_domain::<MyState>("domain");  // Read lock
let mut guard = ctx.get_domain_mut::<MyState>("domain");  // Write lock
```

### Error Propagation

Errors propagate through layers:

```
Domain Code Error
  ↓
Converted to FfiError
  ↓
Wrapped in FfiResult<T>
  ↓
Serialized to JSON ("error:{...}")
  ↓
Returned as *mut c_char
  ↓
Dart parses JSON and creates DartError
```

## Adding New FFI Exports

### Step 1: Define Domain State

```rust
// core/src/ffi/domains/my_domain.rs

#[derive(Debug)]
pub struct MyDomainState {
    pub config: MyConfig,
    pub active: bool,
    pub results: Vec<MyResult>,
}
```

### Step 2: Implement FfiExportable

```rust
use crate::ffi::{FfiExportable, FfiContext, FfiError};

pub struct MyDomainFfi;

impl FfiExportable for MyDomainFfi {
    const DOMAIN: &'static str = "my_domain";

    fn init(ctx: &mut FfiContext) -> Result<(), FfiError> {
        if ctx.has_domain(Self::DOMAIN) {
            return Err(FfiError::invalid_input(
                "my_domain already initialized"
            ));
        }

        ctx.set_domain(Self::DOMAIN, MyDomainState {
            config: MyConfig::default(),
            active: false,
            results: Vec::new(),
        });

        Ok(())
    }

    fn cleanup(ctx: &mut FfiContext) {
        // Stop any background tasks
        // Release resources
        ctx.remove_domain(Self::DOMAIN);
    }
}
```

### Step 3: Add Domain Methods with #[ffi_export]

```rust
use keyrx_ffi_macros::ffi_export;
use crate::ffi::FfiResult;
use serde::Serialize;

#[derive(Serialize)]
pub struct MyResult {
    pub success: bool,
    pub data: Option<String>,
}

impl MyDomainFfi {
    #[ffi_export]
    fn do_something(
        param1: &str,
        param2: u32,
    ) -> FfiResult<MyResult> {
        // Access domain state through global context
        // (handle-based access will be added in future refactor)
        let ctx = global_my_domain_context();
        let mut guard = ctx.lock().unwrap();
        let ctx = guard.as_mut().unwrap();

        let mut state = ctx
            .get_domain_mut::<MyDomainState>(Self::DOMAIN)
            .ok_or_else(|| FfiError::internal("domain not initialized"))?;

        let state = state
            .downcast_mut::<MyDomainState>()
            .ok_or_else(|| FfiError::internal("state type mismatch"))?;

        // Business logic
        state.active = true;

        Ok(MyResult {
            success: true,
            data: Some(format!("Processed: {} ({})", param1, param2)),
        })
    }
}

// Temporary global context (will be replaced with handle-based access)
use std::sync::{Mutex, OnceLock};

fn global_my_domain_context() -> &'static Mutex<Option<FfiContext>> {
    static CONTEXT: OnceLock<Mutex<Option<FfiContext>>> = OnceLock::new();
    CONTEXT.get_or_init(|| Mutex::new(Some(FfiContext::new())))
}
```

class MyDomainBindings {
  final DynamicLibrary _lib;

  MyDomainBindings(this._lib);

  String keyrxDoSomething(String param1, int param2) {
    final param1Ptr = param1.toNativeUtf8();
    try {

## Error Handling

### Error Categories

**INVALID_INPUT**: Client provided invalid data
```rust
FfiError::invalid_input("device_id must not be empty")
FfiError::invalid_input("rows must be between 1 and 20")
```

**INVALID_UTF8**: String parameter contains invalid UTF-8
```rust
FfiError::invalid_utf8("device_id")
// Auto-generated by #[ffi_export] macro
```

**NULL_POINTER**: Null pointer passed for required parameter
```rust
FfiError::null_pointer("device_id")
// Auto-generated by #[ffi_export] macro
```

**NOT_FOUND**: Requested resource doesn't exist
```rust
FfiError::not_found("device")
FfiError::not_found("session")
```

**INTERNAL_ERROR**: Unexpected failure or panic
```rust
FfiError::internal("panic in start_discovery: thread panicked")
FfiError::internal("state type mismatch")
```

### Error Handling Pattern

**In Rust Domain Code**:
```rust
fn my_method(param: &str) -> FfiResult<MyResult> {
    // Validate input
    if param.is_empty() {
        return Err(FfiError::invalid_input("param must not be empty"));
    }

    // Access state
    let state = get_state().ok_or_else(||
        FfiError::internal("domain not initialized")
    )?;

    // Perform operation
    let result = perform_operation(param).map_err(|e|
        FfiError::internal(format!("operation failed: {}", e))
    )?;

    Ok(MyResult { success: true })
}
```

**In Dart**:
```dart
Future<MyResult> doSomething(String param) async {
  final resultJson = keyrxDoSomething(param);

  if (resultJson.startsWith('ok:')) {
    final json = jsonDecode(resultJson.substring(3));
    return MyResult.fromJson(json);
  } else if (resultJson.startsWith('error:')) {
    final json = jsonDecode(resultJson.substring(6));
    throw FfiException(
      code: json['code'],
      message: json['message'],
      details: json['details'],
    );
  } else {
    throw FfiException(
      code: 'PARSE_ERROR',
      message: 'Invalid FFI result format',
    );
  }
}
```

### Panic Safety

All panics are caught at the FFI boundary by the `#[ffi_export]` macro:

```rust
// This panic is caught and converted to FfiError
#[ffi_export]
fn panicking_function() -> FfiResult<()> {
    panic!("Something went wrong!");
    Ok(())
}

// Returns: "error:{\"code\":\"INTERNAL_ERROR\",\"message\":\"panic in panicking_function: Something went wrong!\"}"
```

**Panic Logging**:
Panics are logged to stderr for debugging:
```
[FFI PANIC] Function: panicking_function, Message: Something went wrong!
```

## Best Practices

### 1. State Management

**DO**: Use FfiContext for state storage
```rust
ctx.set_domain(Self::DOMAIN, MyState { ... });
let state = ctx.get_domain::<MyState>(Self::DOMAIN);
```

**DON'T**: Use global statics
```rust
// ❌ Avoid this
static GLOBAL_STATE: OnceLock<Mutex<MyState>> = OnceLock::new();
```

### 2. Error Handling

**DO**: Return FfiResult from all FFI methods
```rust
fn my_method(param: &str) -> FfiResult<MyResult> {
    validate_param(param)?;
    Ok(MyResult { ... })
}
```

**DON'T**: Panic in FFI code
```rust
// ❌ Avoid this - use Result instead
fn my_method(param: &str) -> MyResult {
    assert!(!param.is_empty());  // Panic!
    MyResult { ... }
}
```

### 3. String Parameters

**DO**: Use `&str` for string parameters
```rust
#[ffi_export]
fn my_method(name: &str) -> FfiResult<()> { ... }
```

**DON'T**: Use `String` (unnecessary allocation)
```rust
// ❌ Less efficient
fn my_method(name: String) -> FfiResult<()> { ... }
```

### 4. Callbacks

**DO**: Use EventRegistry for callbacks
```rust
let registry = global_event_registry();
registry.register(EventType::MyDomainProgress, Some(callback));
registry.invoke(EventType::MyDomainProgress, &progress_data);
```

**DON'T**: Create per-method callback functions
```rust
// ❌ Old pattern - creates callback proliferation
#[no_mangle]
pub extern "C" fn keyrx_on_my_domain_progress(cb: ...) { ... }
```

### 5. Cleanup

**DO**: Clean up resources in FfiExportable::cleanup
```rust
fn cleanup(ctx: &mut FfiContext) {
    // Cancel background tasks
    // Close file handles
    // Release system resources
    ctx.remove_domain(Self::DOMAIN);
}
```

**DON'T**: Leak resources
```rust
// ❌ Missing cleanup
fn cleanup(ctx: &mut FfiContext) {
    ctx.remove_domain(Self::DOMAIN);
    // Forgot to cancel background task!
}
```

### 6. Thread Safety

**DO**: Ensure state types implement Send + Sync
```rust
#[derive(Debug)]
struct MyState {
    data: Arc<Mutex<Data>>,  // Thread-safe
}
```

**DON'T**: Use non-thread-safe types
```rust
// ❌ Rc is not Send
struct MyState {
    data: Rc<RefCell<Data>>,  // Compile error!
}
```

### 7. Type Safety

**DO**: Use strongly-typed results
```rust
#[derive(Serialize)]
struct MyResult {
    success: bool,
    data: Option<String>,
}

fn my_method() -> FfiResult<MyResult> { ... }
```

**DON'T**: Return untyped JSON
```rust
// ❌ Harder to maintain
fn my_method() -> FfiResult<serde_json::Value> { ... }
```

### 8. Documentation

**DO**: Document domain methods
```rust
impl MyDomainFfi {
    /// Start processing with the given parameters.
    ///
    /// # Arguments
    /// * `param1` - Description of param1
    /// * `param2` - Description of param2
    ///
    /// # Returns
    /// Success result with processing outcome
    ///
    /// # Errors
    /// - `INVALID_INPUT` if param1 is empty
    /// - `INTERNAL_ERROR` if processing fails
    #[ffi_export]
    fn start(param1: &str, param2: u32) -> FfiResult<MyResult> { ... }
}
```

### 9. Testing

**DO**: Test domain logic independently of FFI
```rust
#[test]
fn test_business_logic() {
    let result = process_data("input");
    assert!(result.is_ok());
}

#[test]
fn test_ffi_integration() {
    let mut ctx = FfiContext::new();
    MyDomainFfi::init(&mut ctx).unwrap();
    let result = MyDomainFfi::start("input", 42);
    assert!(result.is_ok());
}
```

### 10. Versioning

**DO**: Consider FFI stability
```rust
// Add version field for breaking changes
#[derive(Serialize)]
struct ApiVersion {
    major: u32,
    minor: u32,
    patch: u32,
}

#[ffi_export]
fn get_api_version() -> FfiResult<ApiVersion> {
    Ok(ApiVersion { major: 1, minor: 0, patch: 0 })
}
```

## Common Patterns

### Pattern 1: Stateful Operations

```rust
impl MyDomainFfi {
    #[ffi_export]
    fn start() -> FfiResult<()> {
        let ctx = global_context();
        let mut guard = ctx.lock().unwrap();
        let ctx = guard.as_mut().unwrap();

        let mut state_guard = ctx
            .get_domain_mut::<MyState>(Self::DOMAIN)
            .ok_or_else(|| FfiError::internal("not initialized"))?;

        let state = state_guard
            .downcast_mut::<MyState>()
            .ok_or_else(|| FfiError::internal("type mismatch"))?;

        state.active = true;
        state.start_time = Instant::now();

        Ok(())
    }

    #[ffi_export]
    fn stop() -> FfiResult<()> {
        // Similar pattern to access and modify state
        Ok(())
    }
}
```

### Pattern 2: Progress Callbacks

```rust
impl MyDomainFfi {
    #[ffi_export]
    fn start_operation() -> FfiResult<()> {
        let registry = global_event_registry();

        thread::spawn(move || {
            for i in 0..100 {
                // Emit progress
                registry.invoke(
                    EventType::MyDomainProgress,
                    &json!({ "percent": i, "stage": "processing" })
                );

                // Do work
                thread::sleep(Duration::from_millis(100));
            }

            // Emit completion
            registry.invoke(
                EventType::MyDomainComplete,
                &json!({ "success": true })
            );
        });

        Ok(())
    }
}
```

### Pattern 3: Resource Management

```rust
impl MyDomainFfi {
    #[ffi_export]
    fn open_resource(path: &str) -> FfiResult<u64> {
        let resource = Resource::open(path)
            .map_err(|e| FfiError::internal(format!("open failed: {}", e)))?;

        let ctx = global_context();
        let mut guard = ctx.lock().unwrap();
        let ctx = guard.as_mut().unwrap();

        let mut state_guard = ctx
            .get_domain_mut::<MyState>(Self::DOMAIN)
            .ok_or_else(|| FfiError::internal("not initialized"))?;

        let state = state_guard
            .downcast_mut::<MyState>()
            .ok_or_else(|| FfiError::internal("type mismatch"))?;

        let id = state.next_id;
        state.next_id += 1;
        state.resources.insert(id, resource);

        Ok(id)
    }

    #[ffi_export]
    fn close_resource(id: u64) -> FfiResult<()> {
        let ctx = global_context();
        let mut guard = ctx.lock().unwrap();
        let ctx = guard.as_mut().unwrap();

        let mut state_guard = ctx
            .get_domain_mut::<MyState>(Self::DOMAIN)
            .ok_or_else(|| FfiError::internal("not initialized"))?;

        let state = state_guard
            .downcast_mut::<MyState>()
            .ok_or_else(|| FfiError::internal("type mismatch"))?;

        state.resources.remove(&id)
            .ok_or_else(|| FfiError::not_found("resource"))?;

        Ok(())
    }
}
```

## Migration Guide (Legacy → New Architecture)

### Before (Old Pattern)

```rust
// Global static state
static SESSION: OnceLock<Mutex<Option<Session>>> = OnceLock::new();

// Individual callback registration
#[no_mangle]
pub extern "C" fn keyrx_on_progress(cb: extern "C" fn(...)) { ... }

// Manual FFI wrapper
#[no_mangle]
pub unsafe extern "C" fn keyrx_start(param: *const c_char) -> *mut c_char {
    if param.is_null() { return error_json(); }
    let param = match CStr::from_ptr(param).to_str() {
        Ok(s) => s,
        Err(_) => return error_json(),
    };
    // ... business logic
    result_to_json()
}
```

### After (New Pattern)

```rust
// FfiExportable trait
impl FfiExportable for MyDomainFfi {
    const DOMAIN: &'static str = "my_domain";

    fn init(ctx: &mut FfiContext) -> Result<(), FfiError> {
        ctx.set_domain(Self::DOMAIN, MyState::default());
        Ok(())
    }

    fn cleanup(ctx: &mut FfiContext) {
        ctx.remove_domain(Self::DOMAIN);
    }
}

// EventRegistry for callbacks
registry.register(EventType::MyDomainProgress, Some(callback));

// Macro-generated FFI wrapper
impl MyDomainFfi {
    #[ffi_export]
    fn start(param: &str) -> FfiResult<MyResult> {
        // ... business logic (null checks, UTF-8 validation automatic)
        Ok(MyResult { ... })
    }
}
```

## Troubleshooting

### Issue: "domain not initialized" error

**Cause**: Forgot to call `init()` before using domain methods

**Solution**: Ensure `MyDomainFfi::init(&mut ctx)` is called first

### Issue: "state type mismatch" error

**Cause**: Trying to retrieve state with wrong type

**Solution**: Verify state type matches what was stored:
```rust
ctx.set_domain("domain", StateType { ... });
let state = ctx.get_domain::<StateType>("domain");  // Must match!
```

### Issue: Panic at FFI boundary

**Cause**: Panic in domain code not caught by macro

**Solution**: Ensure method has `#[ffi_export]` attribute

### Issue: Callback not being invoked

**Cause**: EventRegistry callback not registered or wrong EventType

**Solution**:
1. Verify callback registered: `registry.is_registered(EventType::...)`
2. Check EventType matches between registration and invocation
3. Ensure callback pointer is not null

### Issue: Memory leak in FFI strings

**Cause**: Not freeing strings returned from FFI

**Solution**: Call cleanup function:
```dart
final resultPtr = keyrxMyMethod(...);
final result = resultPtr.cast<Utf8>().toDartString();
keyrxFreeString(resultPtr);  // Must free!
```

### Issue: Tests interfering with each other

**Cause**: Using global state instead of FfiContext

**Solution**: Create new `FfiContext` per test, use `#[serial]` if needed
