### Integrations

#### Integration
- **Description:** ThreadLocalState provides safe access to thread-local storage used by Windows hook callbacks. The hook callback (extern system function) cannot capture state, so it uses ThreadLocalState to access the event sender and key state tracking.
- **Frontend Component:** Windows hook callback (low_level_keyboard_proc)
- **Backend Endpoint:** ThreadLocalState methods
- **Data Flow:** Hook callback invoked by Windows → ThreadLocalState::track_key_state() to detect repeats → ThreadLocalState::try_send() to send event → Event reaches main processing loop via channel
