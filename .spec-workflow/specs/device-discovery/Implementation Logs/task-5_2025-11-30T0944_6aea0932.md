### Integrations

#### Integration
- **Description:** New `keyrx discover` subcommand routes clap args to DiscoverCommand, streams events through DiscoverySession, saves via DeviceRegistry/storage, and maps DiscoverExit to exit codes 2 (validation) and 3 (cancel).
- **Frontend Component:** keyrx binary (CLI)
- **Backend Endpoint:** DiscoverCommand using DeviceRegistry::save_profile
- **Data Flow:** User runs keyrx discover → select vendor:product or auto-detect → discovery session emits progress/duplicates → optional confirmation → profile saved to ~/.config/keyrx/devices → exit code reflects success/validation/cancel
