### Integrations

#### Integration
- **Description:** ResilientDriver wraps platform drivers (WindowsInput/LinuxInput) with CircuitBreaker to track failures. When failure threshold is reached, circuit opens and FallbackEngine is activated for passthrough mode.
- **Frontend Component:** ResilientDriver
- **Backend Endpoint:** InputSource trait methods (poll_events, send_output, start, stop)
- **Data Flow:** Driver operation → Check circuit state → If open: activate fallback & fail fast → If closed: call underlying driver → Record success/failure with circuit breaker → If failure causes circuit to open: activate fallback → Otherwise propagate result
